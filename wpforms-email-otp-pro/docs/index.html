1. Introduction

1 Overview

WPForms Email OTP Pro enhances your WPForms forms by adding email-based One-Time Password (OTP) verification. Users must verify their email with an OTP before submitting a form, improving security. Key features include:

  • Support for multiple WPForms forms with different email field labels.
  • Automatic removal of OTP field and Send OTP button after successful verification.
  • Customizable OTP emails using Elementor templates.
  • Gmail SMTP integration for reliable email delivery.
  • Configurable OTP length (4, 6, or 8 digits) and expiry time.

2. Installation

1 Upload the Plugin

  1. Go to WordPress Admin → Plugins → Add New.
  2. Click Upload Plugin at the top.
  3. Select the wpforms-email-otp-pro.zip file.
  4. Click Install Now.

2 Activate the Plugin

After installation, click Activate Plugin. If WPForms is not active, you’ll see an error: “WPForms Email OTP Pro requires WPForms.” Install and activate WPForms first.

3. Configuration

1 Access Plugin Settings

In the WordPress admin sidebar, click Email OTP Settings (mail icon).

2 Configure Email Settings

  1. Sender Gmail Address: Enter your Gmail address (e.g., yourname@gmail.com).
  2. Gmail App Password: Paste your Gmail App Password (see Gmail Setup).
  3. Enable SMTP: Check this to use Gmail’s SMTP server.

3 Configure Forms

  1. WPForms Forms and Email Fields: Add rows for each form:
    • Select a form from the dropdown (e.g., “Contact Form (ID: 62)”).
    • Enter the email field label (e.g., Email Address, case-sensitive).
  2. Click Add Form to configure additional forms with different labels (e.g., User Email).
  3. Click Remove to delete a form row (at least one required).

4 Configure OTP Settings

  1. OTP Length: Choose 4, 6, or 8 digits.
  2. OTP Expiry: Set expiry time in minutes (e.g., 5).
  3. Email Subject: Enter the email subject (e.g., Your Verification Code).
  4. Email Message: Customize the email body (e.g., Your verification code is: {otp}), using placeholders {otp} and {site}.

5 Save Settings

Click Save Settings. Refresh to confirm settings are saved.

4. Gmail App Password Setup

1 Enable 2-Step Verification

  1. Go to your Google Account.
  2. Navigate to Security → 2-Step Verification.
  3. Follow the steps to enable 2-Step Verification.

2 Generate App Password

  1. In your Google Account, go to Security → App Passwords.
  2. Select Mail as the app.
  3. Select Other (Custom name) and enter “WPForms OTP”.
  4. Click Generate.
  5. Copy the 16-character password.
Important: The password is shown only once. Store it securely.

3 Enter App Password

Paste the password in the Gmail App Password field in the plugin settings.

5. Form Setup

1 Create or Edit Forms

  1. Go to WPForms → All Forms.
  2. Create a new form or edit an existing one (e.g., ID 62).
  3. Add an Email field with the exact label from settings (e.g., Email Address).
  4. Save the form.
  5. Repeat for each configured form (e.g., another form with User Email).

2 Embed Forms

  1. Add each form to a page or post:
    • Shortcode: Use [wpforms id="62"].
    • Elementor: Add a WPForms widget and select the form.
    • Gutenberg: Use the WPForms block and select the form.
  2. Publish or update the page.

3 User Flow

  1. User enters their email.
  2. Clicks Send OTP.
  3. Receives an OTP email.
  4. Enters the OTP and clicks Verify OTP.
  5. If correct, the OTP field and button disappear, and the Submit button enables.
  6. User submits the form.

6. Custom Email Templates with Elementor

1 Create an Elementor Template

  1. Go to Templates → Saved Templates.
  2. Click Add New, select Section, and name it (e.g., “OTP Email Template”).
  3. Design the email in Elementor:
    • Use a 600px single-column layout.
    • Add text like Your verification code is: {otp} and From: {site}.
    • Use inline CSS for email client compatibility.
  4. Click Publish.

2 Get Template ID

  1. Go to Templates → Saved Templates.
  2. Hover over the template to see the URL (e.g., post=123, where 123 is the ID).
  3. Note the ID.

3 Assign Template

  1. In Email OTP Settings, enter the ID in Elementor Template ID.
  2. Save settings.

7. Testing OTP Functionality

1 Visit a Form

Open a page with an embedded form (e.g., http://your-site.com/form-page). Enter a valid email.

2 Send OTP

Click Send OTP. The OTP field appears. Check your email for the OTP (or MailHog at http://localhost:8025 in a local setup).

3 Verify OTP

Enter the OTP and click Verify OTP. If correct, the OTP field and button disappear, and the Submit button enables.

4 Submit Form

Complete the form and click Submit. Ensure submission succeeds.

5 Test All Forms

Repeat for each configured form to verify different email labels work.

8. Troubleshooting

Emails Not Sending

  • Verify Gmail App Password and Enable SMTP.
  • Ensure 2-Step Verification is enabled in your Google Account.
  • In XAMPP, use MailHog:
    • Run MailHog and access http://localhost:8025.
    • Edit php.ini: 
      [mail function]
SMTP = localhost
smtp_port = 1025
    • Restart XAMPP.
  • Test on a live server for real email delivery.

OTP Verification Not Working

  • Ensure email field labels match settings exactly (case-sensitive).
  • Verify form IDs are correct.
  • Check browser console (F12 → Console) for JavaScript errors.

OTP Field Not Removing

  • Confirm OTP verification succeeds (check AJAX response in console).
  • Ensure assets/js/otp.js is loaded.

Elementor Template Not Loading

  • Ensure Elementor is active.
  • Verify the template ID.
  • Check debug.log for errors.

Enable Debugging

Edit wp-config.php:

define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);

Check wp-content/debug.log for errors.
